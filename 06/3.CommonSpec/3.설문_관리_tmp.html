설문 관리 화면 명세서

1. 설문 생성 관리

  • 기능 요건 모두 충족
  • API를 고려한 버튼 등 이벤트 구성
  • 테이블 컬럼을 고려한 필드명

기능 요건

  • 설문 등록, 조회, 수정, 삭제 기능
  • 설문 유형별(과정 설문, 강사 설문, 일반 설문 등) 관리
  • 설문 기간 설정 및 활성화/비활성화 관리
  • 익명/필수 설문 여부 설정

API

API 명 HTTP 메서드 엔드포인트 설명
설문 등록 POST /api/v1/surveys 설문 등록
설문 조회 GET /api/v1/surveys/{survey_id} 설문 정보 조회
설문 목록 조회 GET /api/v1/surveys 설문 목록 조회
설문 수정 PUT /api/v1/surveys/{survey_id} 설문 정보 수정
설문 삭제 DELETE /api/v1/surveys/{survey_id} 설문 삭제

관련 테이블

  • TB_SURVEY: 설문 정보
    • survey_id: 설문 ID (PK, INT)
    • survey_name: 설문명 (VARCHAR(200))
    • survey_desc: 설문 설명 (TEXT)
    • survey_type: 설문 유형 (VARCHAR(50))
    • course_id: 과정 ID (FK -> TB_COURSE.course_id, INT)
    • class_id: 차수 ID (FK -> TB_COURSE_CLASS.class_id, INT)
    • start_date: 시작일 (DATETIME)
    • end_date: 종료일 (DATETIME)
    • is_anonymous: 익명 여부 (TINYINT(1))
    • is_required: 필수 여부 (TINYINT(1))
    • is_active: 활성 여부 (TINYINT(1))
    • reg_user_id: 등록자 ID (VARCHAR(50))
    • reg_date: 등록일 (DATETIME)
    • mod_user_id: 수정자 ID (VARCHAR(50))
    • mod_date: 수정일 (DATETIME)

2. 설문 문항 관리

  • 기능 요건 모두 충족
  • API를 고려한 버튼 등 이벤트 구성
  • 테이블 컬럼을 고려한 필드명

기능 요건

  • 설문 문항 등록, 조회, 수정, 삭제 기능
  • 문항 유형(객관식 단일, 객관식 복수, 주관식 등) 관리
  • 필수 응답 여부 설정
  • 문항 순서 관리

API

API 명 HTTP 메서드 엔드포인트 설명
설문 문항 등록 POST /api/v1/surveys/{survey_id}/questions 설문 문항 등록
설문 문항 목록 조회 GET /api/v1/surveys/{survey_id}/questions 설문 문항 목록 조회
설문 문항 수정 PUT /api/v1/surveys/{survey_id}/questions/{question_id} 설문 문항 수정
설문 문항 삭제 DELETE /api/v1/surveys/{survey_id}/questions/{question_id} 설문 문항 삭제
설문 선택지 등록 POST /api/v1/questions/{question_id}/options 설문 선택지 등록
설문 선택지 목록 조회 GET /api/v1/questions/{question_id}/options 설문 선택지 목록 조회
설문 선택지 수정 PUT /api/v1/questions/{question_id}/options/{option_id} 설문 선택지 수정
설문 선택지 삭제 DELETE /api/v1/questions/{question_id}/options/{option_id} 설문 선택지 삭제

관련 테이블

  • TB_SURVEY_QUESTION: 설문 문항 정보

    • question_id: 문항 ID (PK, INT)
    • survey_id: 설문 ID (FK -> TB_SURVEY.survey_id, INT)
    • question_text: 문항 내용 (TEXT)
    • question_type: 문항 유형 (VARCHAR(50))
    • is_required: 필수 여부 (TINYINT(1))
    • order_no: 정렬 순서 (INT)
    • reg_date: 등록일 (DATETIME)
    • mod_date: 수정일 (DATETIME)

  • TB_SURVEY_OPTION: 설문 선택지 정보

    • option_id: 선택지 ID (PK, INT)
    • question_id: 문항 ID (FK -> TB_SURVEY_QUESTION.question_id, INT)
    • option_text: 선택지 내용 (VARCHAR(500))
    • score: 점수 (INT)
    • order_no: 정렬 순서 (INT)
    • reg_date: 등록일 (DATETIME)
    • mod_date: 수정일 (DATETIME)

3. 설문 응답 관리

  • 기능 요건 모두 충족
  • API를 고려한 버튼 등 이벤트 구성
  • 테이블 컬럼을 고려한 필드명

기능 요건

  • 설문 응답 시작 및 완료 처리
  • 응답 데이터 수집 및 조회
  • 설문별 응답 현황 모니터링
  • 설문 결과 통계 분석

API

API 명 HTTP 메서드 엔드포인트 설명
설문 응답 시작 POST /api/v1/surveys/{survey_id}/responses 설문 응답 시작
설문 응답 조회 GET /api/v1/surveys/{survey_id}/responses/{response_id} 설문 응답 조회
사용자별 설문 응답 목록 조회 GET /api/v1/surveys/responses/user/{user_id} 사용자별 설문 응답 목록 조회
설문별 응답 목록 조회 GET /api/v1/surveys/{survey_id}/responses 설문별 응답 목록 조회
설문 답변 등록 POST /api/v1/responses/{response_id}/answers 설문 답변 등록
설문 답변 목록 조회 GET /api/v1/responses/{response_id}/answers 설문 답변 목록 조회
설문 응답 완료 처리 PATCH /api/v1/responses/{response_id}/complete 설문 응답 완료 처리
설문 결과 통계 조회 GET /api/v1/surveys/{survey_id}/statistics 설문 결과 통계 조회

관련 테이블

  • TB_SURVEY_RESPONSE: 설문 응답 정보

    • response_id: 응답 ID (PK, INT)
    • survey_id: 설문 ID (FK -> TB_SURVEY.survey_id, INT)
    • user_id: 사용자 ID (FK -> TB_USER.user_id, VARCHAR(50))
    • enrollment_id: 수강신청 ID (FK -> TB_ENROLLMENT.enrollment_id, INT)
    • response_date: 응답일 (DATETIME)
    • is_complete: 완료 여부 (TINYINT(1))
    • complete_date: 완료일 (DATETIME)
    • ip_address: IP 주소 (VARCHAR(50))

  • TB_SURVEY_ANSWER: 설문 답변 정보

    • answer_id: 답변 ID (PK, INT)
    • response_id: 응답 ID (FK -> TB_SURVEY_RESPONSE.response_id, INT)
    • question_id: 문항 ID (FK -> TB_SURVEY_QUESTION.question_id, INT)
    • option_id: 선택지 ID (FK -> TB_SURVEY_OPTION.option_id, INT)
    • text_answer: 텍스트 답변 (TEXT)
    • reg_date: 등록일 (DATETIME)
