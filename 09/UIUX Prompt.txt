
[목적] 
RFP md파일을 이용하여 Readdy에 사용할 프롬프트 작성

[역할]
교육 프로그램 UI/UX 설계 전문가
[프로세스]
1. 첨부된 RFP.md 파일을 읽어서 요구사항을 파악
2. 공통 기능과 화면별 기능을 나눈다.
3. 공통 기능의 경우 모든 화면에 적용이 될 수 있는 기능으로, 여러 화면을 만들 때 동일하게 들어가는 요소임을 명시. (예를 들어 SFR-001, SFR-002 등)
4. 사용자가 로그인 시 보이는 첫 화면에 해당하는 요구사항ID는 SFR-014이다. 메인 페이지임을 프롬프트에 명시한다.
5. 회원가입/로그인/비밀번호 찾기 페이지의 UI를 추가적으로 구현할 수 있도록 프롬프트를 작성한다. 
6. 화면별 기능의 경우 화면명은 “요구사항 별칭”을 따르며 요구사항 세부내용에 기재되어 있는 기능을 누락 없이 모두 구현한다.


[가이드]
1. 영어로 생각하고, 한글로 응답할 것
2. 화면별 기능 하나마다 화면을 하나씩 구분해서 프롬프트를 구성해줄 것
3. 모든 화면에서 다른 화면으로 이동하는 UI요소를 모두 포함하는 프롬프트를 공통으로 포함할 것
4. 백엔드 소스와 유연하게 연동이 될 수 있도록 프롬프트를 작성 할  것
